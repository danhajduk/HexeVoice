# Runtime State Policy

HexeVoice should stay clean after normal install and runtime use. Source code,
seed defaults, and small skeleton files belong in git. Host-specific state,
downloaded artifacts, generated caches, logs, secrets, and migration backups do
not.

## Tracked In Git

- Source code, tests, docs, scripts, and static config.
- `config/runtime-dirs.json`, the source of truth for runtime directory creation.
- Runtime `.gitkeep` files that preserve the expected directory skeleton.
- `runtime/openwakeword/models/hexe.tflite`, the small default Hexe wake model.

## Ignored Local Runtime State

These files are mutable local state. They may be created, updated, migrated, or
backed up, but should not be committed:

- `runtime/onboarding_state.json`
- `runtime/endpoint_registry.json`
- `runtime/voice_intents.json`
- `runtime/voice_tts_settings.json`
- `runtime/voice_session_history.json`
- `runtime/rendered_node_ui_pages/*.json`
- `runtime/endpoint_media/**`
- `runtime/migration/backups/**`
- `runtime/logs/**`
- `runtime/sockets/**`
- `runtime/voice_tts/**`
- `runtime/wake_recordings/**`
- `runtime/micro_vad_chunks/**`

## Ignored Downloads And Artifacts

These are downloaded or generated by install/provider/runtime paths:

- `runtime/firmware/**`
- `runtime/piper-tts/models/**`
- `runtime/stt/**`
- extra `runtime/openwakeword/models/**` files, except the tracked default
  `hexe.tflite`

Firmware, Piper voices, STT models, and extra wake models should be restored by
download/sync scripts or copied as operator-managed artifacts, not committed as
local runtime state.

## Migration And Backup Policy

Migration bundles include supported portable state such as endpoint registry,
voice intents, STT/TTS/wake provider settings, and TTS runtime settings.
Runtime logs, generated audio, retained recordings, model caches, firmware
binaries, rendered UI cache files, and endpoint media payloads are not bundled
by default.

Migration backups live under `runtime/migration/backups/` and are ignored.
Backups are operator artifacts and can contain sensitive operational state
depending on the export mode. Do not commit them.

## Secrets

The following files are ignored and should never be committed:

- `scripts/stack.env`
- `scripts/openwakeword.env`
- `scripts/piper-tts.env`
- local onboarding/trust state under `runtime/`

## Status Helper

Use the runtime status helper when the tree looks noisy:

```bash
./scripts/runtime-state-status.sh
```

It separates reviewable source changes from runtime/local mutable state. Source
changes are candidates for review and commit. Runtime/local mutable state should
normally stay uncommitted.
